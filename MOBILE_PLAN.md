# 移动端适配计划

## 约束
- **必须纯手机浏览器可玩，禁止依赖电脑键盘（A/S/D 键、Escape 键）**
- 所有交互必须通过触摸屏完成
- 界面布局需适配竖屏手机（375px-430px 宽度）

## 当前问题
- 节奏游戏核心操作用 A/S/D 三键对应三条轨道，手机无法使用
- Escape 键暂停/菜单，手机无键盘
- 没有任何触摸事件
- 主文件：`js/input-system.js`

## 修改方案

### 1. 三轨道触摸按钮
- `js/input-system.js`：在画面底部添加三个等宽触摸区域/按钮
- 每个按钮对应一条轨道（左/中/右），替代 A/S/D
- 按钮样式：半透明，带轨道颜色标识，高度 80-100px（方便拇指）
- touchstart = 按下（keydown），touchend = 松开（keyup）
- 支持多指同时触摸（三条轨道可同时按下）

### 2. 暂停按钮
- 画面右上角添加暂停图标按钮（⏸ 或 ☰）
- 替代 Escape 键功能
- 暂停后显示菜单：继续、重新开始、返回

### 3. 竖屏节奏布局
- 三条轨道竖向排列，占满屏幕宽度
- 判定线在底部按钮区域上方
- 下落音符从上往下，视觉清晰
- canvas 宽度 100vw，高度 100vh（全屏沉浸）

### 4. 触摸精度优化
- 防止触摸延迟：使用 `touchstart` 而非 `click`
- `{ passive: false }` + `preventDefault()` 防止滚动
- 添加触觉反馈（`navigator.vibrate(10)`）如果浏览器支持
- `touch-action: none` 在游戏区域（防止所有默认手势）

### 5. 保留键盘兼容
- 桌面端仍可用 A/S/D 和 Escape
- 移动端隐藏键盘提示，显示触摸按钮
- 通过 `'ontouchstart' in window` 或 matchMedia 检测设备类型
